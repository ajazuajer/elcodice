En el Lenguaje : Instrucciones y Sintaxis

El diseño de Algoritmos, Diagramas de Flujo de Datos e incluso el Pseudocódigo es la mitad del camino del programador, ya que posterior a esto, se continúa con la escritura de lo producido en un lenguaje de programación.

Sin importar el lenguaje en el que se disponga a trabajar se tienen los siguientes elementos dentro del código fuente:

Conjunto de caracteres: se refiere a los  dígitos del teclado y otros símbolos de los idiomas que se utilizan para escribir un código de programa.

Identificadores: es un conjunto de caracteres alfanuméricos de cualquier longitud que sirve para identificar las entidades del programa.

Símbolos de Operadores: comprende cada uno de los operadores aritméticos, lógicos y relacionales que se implementan en la programación.

Palabras Reservadas: es un identificador que se usa como una parte fija de la sintaxis de un enunciado.

Comentarios: se utilizan para documentar un sistema, su implementación permite al programador conocer los bloques de código.

Expresiones: son bloques sintácticos de construcción a partir de los cuales se construyen enunciados.

Instrucciones: representan cada una de las líneas de código escrito en un programa.

Bucle: en ocasiones un mismo grupo de instrucciones se repite un número determinado de veces, por lo que se le llama bucle a ese grupo determinado de instrucciones.

Finalmente, justo cuando nos disponemos a probar nuestro programa, es decir, “correr” la aplicación, podemos observar como el lenguaje hace lo que hace, de manera que el computador lleva a cabo las siguientes tareas antes de mostrar nuestro programa:

Análisis Léxico: en la mayoría de los últimos lenguajes, el programa evalúa cada instrucción al terminar de escribirla, además, al finalizar de escribir todo el código fuente se realiza un análisis en cuanto a identificadores, operadores, palabras claves, comentarios, etc; de manera que este se encarga de velar porque cada instrucción sea escrita correctamente.

Análisis Sintáctico: se encarga de evaluar todas las declaraciones de variables, estructuras, expresiones, llamadas a procedimientos; de manera que este se encarga de velar porque cada sentencia tenga la estructura correcta para que sea procesada por el compilador.

Vinculación: proceso de enlazado del código fuente con los analizadores del lenguaje de programación.

Compilación: proceso en el cual se traduce el código fuente al lenguaje máquina.

Carga: se refiere cuando el programa se está ejecutando.
