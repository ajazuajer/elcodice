Conceptos de la Programación Orientada a Objetos

Los principales aspectos, (y que suelen ser llamado pilares) en los cuales se basa la POO son la Herencia, Encapsulación y Polimorfismo. Sin embargo, algunos autores difieren y en algunos casos le agregan algunos terminos mas. A continuación se presenta lo que desde el punto de vista del Docente representan la base de este modelo.

Herencia:
Una relación de herencia es una relación en la que un tipo (el tipo derivado) se deriva de otro (el tipo base), de tal forma que el espacio de declaración del tipo derivado contiene implícitamente todos los miembros de tipo no constructor del tipo base.

Según la propia documentación de Visual Studio .NET:
"La herencia permite crear nuevas clases a partir de clases existentes. La herencia puede simplificar el diseño de la aplicación proporcionando una estructura de relaciones entre las distintas clases. También admite la reutilización de código porque sólo se debe codificar el comportamiento de clases nuevas o distintas.".

Encapsulación:
Consiste en ocultar las características internas de un objeto (los datos y el código que los manipula), de forma que sólo se pueda acceder al objeto a través de las partes declaradas como públicas (usando public). Las partes privadas (private) sólo podrán ser accedidas desde el código del propio objeto.

Según Visual Studio.NET:
"La encapsulación es la capacidad de contener y controlar el acceso a un grupo de elementos asociados. Las clases proporcionan una de las formas más comunes de encapsular elementos."
Cuando usamos las clases, éstas tienen una serie de características (los datos que manipula) así como una serie de comportamientos (las acciones a realizar con esos datos). La encapsulación es esa capacidad de la clase de ocultarnos sus interioridades para que sólo veamos lo que tenemos que ver, sin tener que preocuparnos de cómo está codificada para que haga lo que hace... simplemente nos debe importar que lo hace.

Polimorfismo:
Se refiere a la posibilidad de definir múltiples clases con funcionalidad diferente, pero con métodos o propiedades denominados de forma idéntica, que pueden utilizarse de manera intercambiable mediante código cliente en tiempo de ejecución.

Según la documentación de Visual Studio.NET:
"El polimorfismo se refiere a la posibilidad de definir múltiples clases con funcionalidad diferente, pero con métodos o propiedades denominados de forma idéntica, que pueden utilizarse de manera intercambiable mediante código cliente en tiempo de ejecución."

Dicho de otra manera, puede tener múltiples clases que se pueden utilizar de forma intercambiable, si bien cada clase implementa las mismas propiedades o los mismos métodos de maneras diferentes. El polimorfismo es importante en la programación orientada a objetos puesto que permite usar elementos que tienen el mismo nombre, independientemente del tipo de objeto que se esté utilizando en ese momento.

Ahora bien, en cuanto nos dispongamos a escribir nuestro código fuente es necesario entender estos conceptos pues estaremos trabajando con ellos durante todo el tiempo de desarrollo del software:

Objetos y Clases:
Las clases son las unidades de creación de una aplicación orientada a objetos. Una clase es una construcción de programación que representa un objeto en el sistema. Al diseñar una aplicación es necesario diseñar los objetos implicados, la información que contienen los objetos y las operaciones que pueden llevar a cabo los objetos.

Objetos:
Un objeto es una abstracción de algo dentro de un dominio de un problema, que refleja las capacidades del sistema para almacenar información sobre él, para interactuar con él o para ambas cosas.
Clase y objeto es un concepto que describe un conjunto de objetos que se han especificado de la misma forma. De acuerdo con la especificación UML, todos los objetos de una determinada clase comparten una especificación común de sus características, su semántica y las restricciones que existen sobre ellos (OMG, 2004c). Esta definición no significa que todos los objetos de una clase sean idénticos desde todos los puntos de vista posibles, sino que su especificación es la misma. Los objetos que son suficientemente similares entre si se dice que pertenecen a la misma clase y, a su vez, la clase es un descriptor abstracto para las similitudes lógicas especificadas que existen entre estos objetos.

Los programas de Visual Basic se generan con objetos como formularios y controles. Los objetos también se pueden utilizar para representar cosas reales como personas, equipos informáticos o incluso algo más abstracto, como una cuenta bancaria.

Una clase es simplemente una representación de un tipo de objeto.

Espacio de Nombre: es una biblioteca jerárquica de clases organizadas bajo un nombre único, como System.Windows o System.Diagnostics. También se les conoce como biblioteca de objetos o de clases.

Instrucción de Programa: es una línea de código de programa, una instrucción autocontenida y ejecutada por el compilador para realizar un trabajo útil dentro de la aplicación. Si nos referimos a la programación estructurada (anterior a ésta), una instrucción es una línea de código escrita.

Palabra Clave: es una palabra reservada reconocible por el compilador y necesaria para realizar un trabajo útil. Casi todas las palabras claves se muestran con color azul en el editor de código.

Variable: es un contenedor especial que se usa para almacenar datos de manera temporal. Se pueden crear usando la instrucción Dim.

Control: es una herramienta que se usa para crear objetos en los formularios. Se seleccionan en el cuadro de herramientas y se dibujan en el formulario.

Clase: es un plano o plantilla para uno o más objetos; su propósito es definir lo que hará ese objeto. Una clase define lo que puede hacer un objeto (métodos) y las caracteristicas que tendrá (propiedades), pero no es el objeto en sí mismo.

Instancia de una Clase: es la creación de un objeto partiendo de una clase. "Instanciamos"un objeto cuando dimensionamos una variable como una nueva "clase" que hemos diseñado.

Objeto: es una estructura completa que contiene lo definido por una clase en función de sus propiedades y métodos. Tambien suele llamarse objeto a los controles del cuadro de herramientas pues cumplen con esa estructura..

Propiedad: son los valores o características de los objetos los cuales estan establecidos en la clase a la cual pertenecer. Es posible establecer las propiedades tanto en tiempo de diseño como en tiempo de ejecución

Método: es una instrucción especial que realiza una acción o un servicio para un objeto en particular dentro de un programa. Puede decirse que son los procedimientos propios de cada clase, los cuales serán accionados por los objetos de acuerdo a los parametros establecidos en la clase.

Procedimiento de Evento: son bloques de código que se ejecutan cuando un objeto es manipulado en un programa.
